## 🎯 สรุปความต้องการของระบบ (System Goals)
ระบบถูกออกแบบมาเพื่อช่วยแบ่งเบาภาระงานดูแลช่วยเหลือนักเรียนของครูประจำชั้น แผนกช่างกลโรงงาน และแผนกเทคนิคอุตสาหกรรม โดยเน้นประเด็นสำคัญ:
1. **ความพรีเมียม สวยงาม และใช้งานง่าย**: แสดงผลแดชบอร์ดสถิติในรูปแบบโมเดิร์น มีโหมดสีให้เลือกใช้ (Light/Dark Mode)
2. **การคัดกรองความเสี่ยงรายด้าน (SDQ & Risk Screening)**: คัดกรองระดับความเสี่ยง (ปกติ/ปานกลาง/สูง) แยกตามรายด้านเพื่อวางแผนช่วยเหลือ เช่น ด้านการเรียน พฤติกรรม ครอบครัว และเศรษฐกิจ
3. **การนำเข้า/ส่งออกข้อมูล (Cloud Sync & Backup)**: เชื่อมต่อผ่านตาราง Google Sheets และการส่งออกเป็นไฟล์ตาราง CSV ภาษาไทยสำหรับ Excel

## 🛠️ รายการสิ่งสำคัญที่ได้อัปเกรดพัฒนาเพิ่ม (Key Enhancements implemented)

### 1. ระบบชั้นปีแบบไดนามิก (Dynamic Class Years)
* **การแก้ไข**: ปลดล็อกขีดจำกัดเดิมที่ล็อกไว้ปีที่ 1 เพียงอย่างเดียว ให้สอดรับโครงสร้างสถานศึกษาจริง:
  * ระดับ **ปวช. (ช่างกลโรงงาน)** -> รองรับชั้นปีการศึกษา **1, 2, 3**
  * ระดับ **ปวส. (เทคนิคอุตสาหกรรม)** -> รองรับชั้นปีการศึกษา **1, 2**
* **แดชบอร์ด**: ปรับปรุงกราฟแท่งเปรียบเทียบในหน้าหลักให้คำนวณและแสดงสถิติแยกเป็น 5 ประเภทแท่ง (ปวช. 1-3 และ ปวส. 1-2) อย่างสวยงาม
* **ตัวกรองมือถือ**: ขยายแถบ Scroll Chip ตัวกรองทางลัดบนมือถือให้ครอบคลุม 5 ประเภทชั้นปีการศึกษา

### 2. ข้อมูลการฝึกงาน/สหกิจศึกษา (ปวส. เทคนิคอุตสาหกรรม)
* **การแก้ไข**: เพิ่มฟิลด์ **สถานที่ฝึกงาน / สหกิจศึกษา (`internship_place`)** และ **เบอร์โทรศัพท์ติดต่อสถานที่ฝึกงาน (`internship_phone`)**
* **กลไกอัจฉริยะ**: ฟิลด์นี้จะแสดงขึ้นมาให้บันทึกและโชว์ในระเบียบประวัติผู้เรียน เฉพาะเมื่อคลิกเลือกเป็นระดับ **ปวส.** เท่านั้น เพื่อให้หน้าต่าง UI ดูสะอาดเป็นระเบียบที่สุดสำหรับ ปวช.

### 3. ระบบซิงค์ตัวอักษรของกราฟตามโหมดสี (Real-time Chart Theme Sync)
* **การแก้ไข**: พัฒนาฟังก์ชัน `updateChartsTheme()` เข้ามาแก้ไขปัญหาตัวหนังสือบนกราฟอ่านยากเมื่อกดสลับโหมด ☀️/🌙 
* **กลไก**: เมื่อกดสลับระหว่าง Light Mode และ Dark Mode ตัวอักษรแกน กราฟแท่ง กราฟวงกลม และเส้นตารางเบื้องหลัง จะสลับเฉดความเข้ม-สว่าง เพื่อให้กลมกลืนและมีความเปรียบต่างที่คมชัดและสวยงามตามโหมดที่ใช้อยู่จริงโดยอัตโนมัติ

### 4. ระบบวิเคราะห์และดึงรูปภาพประจำตัวผู้เรียนอัตโนมัติ (Intelligent Photo Syncer)
* **ความต้องการ**: ดึงรูปภาพจากลิงก์ส่งงาน Google Drive ของนักเรียนใน Google Sheets มาจับคู่เข้าโปรไฟล์นักเรียนตามชื่อและรหัสประจำตัว
* **คุณสมบัติเด่นของ Import Engine**:
  * **Header Collision Guard**: สแกนข้อมูลลงไปในแถวถัดไปเพื่อหาคอลัมน์ที่เป็นไฟล์ลิงก์ภาพ Google Drive อัตโนมัติ ป้องกันคอลัมน์ชื่อซ้ำกัน
  * **Double-Check Matcher**: จับคู่ด้วยรหัสนักเรียนเป็นอันดับแรก หากรหัสว่าง จะทำความสะอาดคำนำหน้าชื่อ (`นาย/นางสาว`) และ Whitespace เพื่อใช้สแกนหาผู้เรียนที่ชื่อตรงกันแทน
  * **Drive to CDN Link Converter**: แปลงลิงก์แชร์เปิดดูธรรมดาของ Google Drive ให้กลายเป็นลิงก์เรียกภาพจาก Google CDN Direct Link ทำให้อวาตาร์แสดงขึ้นเป็นรูปภาพขนาดจริงได้ทันทีบนเบราว์เซอร์
  * **Local CSV Photo Sync (CORS Shield)**: เพิ่มช่องทางสำรองให้คุณครูดาวน์โหลดชีตเป็นไฟล์ `.csv` ลงเครื่องแล้วกดเลือกไฟล์มาอัปโหลดซิงค์ภาพได้ทันที เพื่อป้องกันปัญหาความปลอดภัยของเบราว์เซอร์บล็อกการยิงขอภาพ (CORS Block)

## 🐛 บันทึกการแก้ไขบั๊กสำคัญ (Bug Fixes Log)

### 🔴 บั๊ก Type Coercion ในการแก้ไขประวัติ
* **อาการ**: เมื่อคลิกปุ่ม "แก้ไข" หรือเรียกใช้ข้อมูลจากฐานข้อมูล ระบบไม่แสดงหน้าต่างและแก้ไขประวัติไม่ได้
* **สาเหตุ**: ชนิดข้อมูลของรหัส ID จากฐานข้อมูลเป็นตัวเลข (Number เช่น `662010101`) แต่พารามิเตอร์ที่หน้าเว็บส่งออกไปเป็นข้อความ (String เช่น `"662010101"`) เมื่อใช้การเช็กแบบ `===` จึงทำให้หาข้อมูลนักเรียนไม่เจอ
* **การแก้ไข**: ปรับปรุงคำสั่งใน `app.js` ของฟังก์ชัน `openEditModal()`, `saveStudent()`, `deleteStudent()`, `viewProfile()` และ `printProfile()` ให้แปลงชนิดข้อมูลเป็น String ก่อนทำการเทียบกันเสมอ (`String(x.id) === String(studentId)`) ทำให้แก้ไขและอัปเดตข้อมูลลงฐานข้อมูลเบราว์เซอร์ได้สมบูรณ์เสถียร 100%

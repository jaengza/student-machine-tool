# แผนการพัฒนาระบบซิงค์ข้อมูลและบันทึกผลเรียลไทม์ข้ามเครื่อง (Google Sheets Real-time Cloud API)

เราจะยกระดับระบบฐานข้อมูลนักเรียนและคัดกรองความเสี่ยง จากเดิมที่เป็นระบบความจำเฉพาะเครื่อง (LocalStorage) ให้กลายเป็น **ระบบคลาวด์ส่วนตัวที่แชร์ข้อมูลสดเรียลไทม์ตรงกันในทุกอุปกรณ์ข้ามโลก** (คอมพิวเตอร์และมือถือ) โดยใช้ **Google Sheets** แผ่นเดิมของคุณครูเป็นฐานข้อมูลหลักผ่านบริการ **Google Apps Script**

---

## 🔒 แนวคิดและสถาปัตยกรรมคลาวด์ (Cloud Architecture)

* **Google Sheets เป็น Backend Databaseหลัก**: ข้อมูลนักเรียนทั้งหมด (ชื่อ, รหัส, นามสกุล, เบอร์โทร, รูปภาพ, ข้อมูลฝึกงาน และผลการประเมินความเสี่ยงทั้งหมด) จะจัดเก็บอยู่บน Google Sheets ของคุณครูโดยตรง
* **Google Apps Script เป็นตัวเชื่อมระบบ (Cloud API Gateway)**: เราจะฝังชุดคำสั่ง Apps Script เล็ก ๆ ไว้ใน Google Sheets เพื่อทำหน้าที่รับส่งข้อมูลผ่านทางหน้าเว็บ (รับ Request แก้ไขข้อมูลจากเว็บ และเขียนข้อมูลกลับลงชีตทันที)
* **การบันทึกข้ามเครื่องสด (Real-time Sync & Save)**:
  * เมื่อคุณครูกรอกข้อมูลประเมินหรือแก้ไขข้อมูลนักเรียนในคอมพิวเตอร์ -> ข้อมูลจะถูกเขียนกลับลงไปใน Google Sheets ทันที
  * เมื่อเปิดหน้าเว็บในโทรศัพท์มือถือ -> หน้าเว็บจะทำการดึงข้อมูลล่าสุดจาก Google Sheets มาแสดงผลสด ๆ เสมอ!
  * **ไม่มีการสูญหายของข้อมูล**: ระบบจะทำการบันทึกและซิงค์ผ่านอินเทอร์เน็ตทุกครั้งที่มีการกดบันทึก

## 🛠️ รายละเอียดการทำงานและสิ่งที่จะพัฒนา

### 1. โค้ดหลังบ้าน (Google Apps Script Backend Code)
เราจะสร้างชุดโค้ด JavaScript พิเศษ (Apps Script) เพื่อนำไปแปะหลัง Google Sheets ของคุณครู โดยมีขีดความสามารถ:
* ฟังก์ชัน `doGet(e)`: ทำหน้าที่อ่านข้อมูลแถวและคอลัมน์ทั้งหมดในชีต ส่งกลับมาเป็นข้อมูลรูปแบบ JSON ให้หน้าเว็บแสดงผล
* ฟังก์ชัน `doPost(e)`: ทำหน้าที่รับข้อมูลการแก้ไขของนักเรียนจากหน้าเว็บ ค้นหาแถวของรหัสนักเรียนคนนั้นในชีต และทำการ**เขียนบันทึกปรับปรุงข้อมูลช่องนั้น ๆ กลับคืนสู่ Google Sheets ทันที!**

### 2. โค้ดหน้าเว็บฝั่งไคลเอนต์ (Web App Front-end Integrations)
* **📄 [index.html](file:///G:/My%20Drive/student-database/index.html)**:
  * เพิ่มช่องกรอกข้อมูล **"ลิงก์ Web App API ของ Google Sheets"** ในเมนูตั้งค่าระบบ เพื่อใช้ระบุช่องทางการเชื่อมต่อ
* **📄 [app.js](file:///G:/My%20Drive/student-database/app.js)**:
  * พัฒนาระบบตรวจสอบตัวเลือก: หากมีการกำหนดลิงก์ Web App API ไว้ ระบบจะปรับการโหลดและเซฟฐานข้อมูลจากแบบ Local Storage มาเป็นแบบ **คลาวด์ออนไลน์ (Cloud Mode)** โดยอัตโนมัติ
  * ปรับปรุงฟังก์ชันบันทึกข้อมูล `saveStudent()`: หากเป็นโหมดออนไลน์ ระบบจะส่ง Request ไปเซฟบนคลาวด์ Google Sheets หลังบ้านทันทีผ่านคำสั่ง `fetch()`
  * ปรับปรุงระบบโหลดฐานข้อมูล `loadDatabase()`: ดึงข้อมูลสดจากคลาวด์ Google Sheets มาแสดงผลเสมอเมื่อเปิดเว็บ

## 📋 ขั้นตอนความร่วมมือ (5 ขั้นตอนง่าย ๆ สำหรับคุณครู)

เพื่อให้ระบบทำงานได้สำเร็จตามแผนงานนี้ คุณครูและผมจะร่วมมือกันทำดังนี้ครับ:

1. **ผมเขียนโค้ดและเตรียมชุดคำสั่ง**: ผมจะรวบรวมเขียนโค้ด Apps Script ทั้งหมดไว้ในระบบ และแสดงคู่มือการใช้งานให้คุณครูเห็นอย่างชัดเจนในแชทนี้
2. **คุณครูเปิดระบบ Apps Script ใน Google Sheets**:
   * คุณครูเปิด Google Sheets แผ่นข้อมูลนักเรียนหลักของตนเองขึ้นมา
   * มองหาแถบเมนูด้านบน แล้วคลิกปุ่ม **ส่วนขยาย (Extensions) > Apps Script**
3. **คุณครูวางโค้ดและบันทึก**:
   * นำโค้ดที่ผมเตรียมไว้ให้ ไปวางทับหน้ากระดาษว่าง ๆ ใน Apps Script
   * กดปุ่มไอคอน **บันทึก (Save)** (รูปแผ่นดิสก์)
4. **คุณครูเผยแพร่แอป (Deploy)**:
   * คลิกปุ่มสีฟ้าขวาบนชื่อ **การทำให้ใช้งานได้ (Deploy) > การทำให้ใช้งานได้ใหม่ (New deployment)**
   * ตรงตัวเลือกประเภท (Gear icon) ให้เลือกเป็น **เว็บแอป (Web app)**
   * ตรงตัวเลือก *"ผู้มีสิทธิ์เข้าถึง (Who has access)"* ให้คลิกเลือกเป็น **ทุกคน (Anyone)** *(สำคัญมาก เพื่อให้หน้าเว็บส่งข้อมูลเข้าได้)*
   * คลิกปุ่ม **การทำให้ใช้งานได้ (Deploy)** และกดอนุญาตสิทธิ์การเข้าถึง (Authorize Access) ผ่านอีเมล Google ของคุณครู
5. **นำลิงก์มาใช้**: คุณครูจะได้ลิงก์ยาว ๆ ที่เรียกว่า **URL ของเว็บแอป (Web App URL)** ให้นำลิงก์นั้นมาวางในหน้าเว็บของเรา เพื่อเปลี่ยนเว็บเป็นระบบเชื่อมต่อสดออนไลน์ทันทีครับ!

## 🧪 แผนการตรวจสอบและทดสอบ (Verification Plan)

1. **ทดสอบการซิงค์ข้อมูลเริ่มต้น (Read Test)**: เปิดหน้าเว็บในเครื่องคอมพิวเตอร์ ระบบต้องดึงรายชื่อและข้อมูลคัดกรองทั้งหมดมาจาก Google Sheets ได้สำเร็จโดยไม่มีข้อผิดพลาด
2. **ทดสอบการอัปเดตข้อมูล (Write Test)**: กดแก้ไขเบอร์โทรศัพท์หรือความเสี่ยงของนักเรียน 1 คนในระบบหน้าเว็บ แล้วไปเปิดดูใน Google Sheets ของจริง ข้อมูลแถวนั้นต้องอัปเดตเปลี่ยนค่าตามสด ๆ ทันที!
3. **ทดสอบใช้งานข้ามอุปกรณ์ (Cross-device Sync Test)**: เปิดหน้าเว็บในมือถือ ป้อนรหัสผ่าน และดูข้อมูลความเสี่ยงนักเรียนคนที่เพิ่งแก้ไขไปในคอมพิวเตอร์ ข้อมูลต้องอัปเดตตรงกันโดยไม่ต้องกดส่งออกไฟล์ CSV ใด ๆ เลย!

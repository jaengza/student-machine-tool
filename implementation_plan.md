# แผนการพัฒนาระบบ Cloud-First เรียลไทม์เป็นปัจจุบันตลอดเวลา (Real-time Always Up-to-Date v8.0)

เพื่อตอบสนองความต้องการของคุณครูที่ต้องการให้โน้ตบุ๊กทั้ง 2 เครื่อง รวมถึงสมาร์ทโฟนมือถือทุกเครื่องที่เข้าใช้งาน สามารถอัปเดต แก้ไข และบันทึกข้อมูลนักเรียนได้อย่างสอดคล้อง มีความสมบูรณ์และเป็นปัจจุบันตลอดเวลา (Always Up-to-Date) เราจะยกระดับระบบให้เป็นสถาปัตยกรรมแบบ **Cloud-First** อย่างแท้จริง โดยเสริมสร้างระบบดังนี้ครับ:

---

## 🔒 ฟีเจอร์หลักในการยกระดับระบบ (Key Feature Highlights)

### 1. ระบบลบข้อมูลออนไลน์ (Cloud Delete API)
* **ปัญหาเดิม**: เมื่อลบประวัตินักเรียนผ่านหน้าเว็บ ข้อมูลจะถูกลบแค่ใน Local Storage ของเครื่องนั้น แต่แถวข้อมูลบน Google Sheets ยังคงอยู่ ส่งผลให้อีกเครื่องหนึ่งพอดึงข้อมูลใหม่นักเรียนคนนั้นจะฟื้นกลับขึ้นมาแสดงผลเหมือนเดิม
* **แนวทางแก้ไข**: 
  * ปรับปรุงสคริปต์ Google Apps Script (`doPost`) บนกูเกิลชีตหลังบ้าน ให้รองรับพารามิเตอร์คำสั่งลบ `"action": "delete"`
  * เมื่อได้รับพารามิเตอร์ลบ สคริปต์จะค้นหารหัสประจำตัวของนักเรียนคนนั้น แล้วลบแถวดิจิทัลทั้งหมดออกจาก Google Sheets ของจริงทันที!
  * ปรับแต่งหน้าเว็บในฟังก์ชัน `deleteStudent()` ให้ทำการเรียกส่งคำสั่งไปลบบนคลาวด์เช่นเดียวกัน

### 2. ระบบดึงประวัติสดทันทีเมื่อสลับกลับมาหน้าจอ (Sync on Focus)
* **ปัญหาเดิม**: คุณครูเปิดหน้าเว็บค้างไว้บนโน้ตบุ๊กทั้ง 2 เครื่อง เครื่องหนึ่งมีการพิมพ์แก้ไขข้อมูล อีกเครื่องจะไม่ได้รับรู้ความเคลื่อนไหวใด ๆ เลยจนกว่าจะกดรีเฟรชหน้าจอเองด้วยมือ
* **แนวทางแก้ไข**:
  * ฝังกลไกเหตุการณ์ `window.addEventListener('focus', ...)`
  * ทันทีที่คุณครูสลับหน้าต่างกลับมาที่เบราว์เซอร์นี้ หรือเปิดโน้ตบุ๊กอีกเครื่องขึ้นมาดูหน้าจอ ระบบจะสั่งดาวน์โหลดดึงประวัติล่าสุดออนไลน์เบื้องหลัง (Background Fetch) มาอัปเดตให้เป็นตัวล่าสุดทันทีโดยไม่มีการสะดุด!

### 3. ระบบซิงค์สดเบื้องหลังอัตโนมัติ (Background Auto-Sync Engine)
* **แนวทางแก้ไข**:
  * ฝังกลไกสั่งการทำงานเป็นเวลา `setInterval` คอยตรวจเช็กดึงข้อมูลล่าสุดจาก Google Sheets เบื้องหลังแบบเงียบ ๆ ทุก ๆ 60 วินาที
  * **เกราะกันฟอร์มสะดุด (Active Form Guard)**: ระบบจะตรวจสอบก่อนว่าคุณครูกำลังพิมพ์หรือเปิดหน้าต่าง Modal แก้ไขประวัตินักเรียนค้างอยู่หรือไม่ หากครูกำลังทำงานคีย์ประวัติอยู่ ระบบจะยกเลิกการซิงค์รอบนั้นอัตโนมัติ เพื่อป้องกันไม่ให้ข้อมูลที่คุณครูกำลังคีย์อยู่โดนฐานข้อมูลใหม่เขียนทับสูญหายไป!

## 🛠️ รายละเอียดสิ่งที่จะปรับปรุงในแต่ละไฟล์

### 1. [google_apps_script.js](file:///G:/My%20Drive/student-database/google_apps_script.js) (สคริปต์หลังชีต)
* เพิ่มการรับพารามิเตอร์ `action` ภายในฟังก์ชัน `doPost(e)`
* หากพบว่าพารามิเตอร์ `action === "delete"` ให้รันคำสั่งค้นหาแถวและลบแถว `sheet.deleteRow(targetRowIdx)` ทันที
* ส่งผลตอบรับสถานะการลบแถวกลับมายังหน้าเว็บแอป

### 2. [app.js](file:///G:/My%20Drive/student-database/app.js) (ตัวสมองหน้าเว็บ)
* **ปรับปรุง `deleteStudent()`**: พัฒนาให้ทำการส่ง Request POST แบบ `"action": "delete"` ไปยัง Cloud API เมื่อผู้ใช้ตอบตกลงลบข้อมูล
* **สร้างฟังก์ชัน `syncDatabaseBackground()`**: เป็นตัวประสานข้อมูลหลังบ้าน ดึงข้อมูลจาก Cloud เงียบ ๆ และสแกนตรวจสอบว่าข้อมูลใหม่กับข้อมูลเดิมในเครื่องมีความต่างกันหรือไม่ หากต่างกันและไม่ได้เปิด Form แก้ไขค้างอยู่ ให้แอบแปลงข้อมูลและอัปเดตหน้าจอทันทีอย่างไหลลื่น
* **สร้างฟังก์ชัน `startAlwaysUpToDateEngine()`**: ติดตั้งตัวดักจับ Windows Focus Event และตั้งเวลาทำงาน (setInterval) ทุก ๆ 60 วินาที เพื่อสร้างวงล้อการซิงค์ข้อมูลให้เป็นปัจจุบันตลอดเวลา

## 📋 ขั้นตอนการปรับปรุงระบบและการทำงานร่วมกัน

1. **ผู้ใช้ยินยอมอนุมัติแผน**: เพื่อป้องกันข้อผิดพลาดเชิงโครงสร้าง
2. **ผมทำการอัปเดตโค้ดระบบ**: ลงมือเขียนโค้ดปรับปรุงในไฟล์ `google_apps_script.js` และ `app.js` บน Google Drive
3. **คุณครูอัปเดตโค้ดสคริปต์หลัง Google Sheets**:
   * เปิด Google Sheets แล้วไปที่ **ส่วนขยาย (Extensions) > Apps Script**
   * ก๊อปปี้สคริปต์ตัวใหม่ใน `google_apps_script.js` ไปวางทับตัวเดิม และกดบันทึก
   * **สำคัญ**: ทำการ **เผยแพร่การใช้งานใหม่ (Deploy > Manage deployments > Edit > New version > Deploy)** เพื่ออัปเดต API บนชีตให้เป็นเวอร์ชันรองรับการลบข้อมูล
4. **พร้อมใช้งานข้ามอุปกรณ์**: โน้ตบุ๊กทั้ง 2 เครื่องและสมาร์ทโฟนจะได้รับสิทธิ์การอัปเดตสดพร้อมกันทันที!

## 🧪 แผนการตรวจสอบและทดสอบความสอดคล้อง (Verification Plan)

1. **การตรวจสอบการลบข้อมูล (Delete Test)**: ทดลองกดปุ่มลบนักเรียนจำลอง 1 คนบนโน้ตบุ๊กเครื่องที่ 1 และเปิดดูตารางใน Google Sheets ของจริงว่า แถวข้อมูลนั้นถูกลบออกไปจริง ๆ หรือไม่
2. **การตรวจสอบการสลับหน้าจอ (Sync-on-Focus Test)**:
   * เปิดหน้าเว็บบนโน้ตบุ๊กทั้ง 2 เครื่องค้างไว้
   * ที่เครื่องที่ 1 แก้ไขชื่อเล่นหรือสถิติเสี่ยงของนักเรียนแล้วกดบันทึก
   * สลับไปมองที่เครื่องที่ 2 แล้วขยับเมาส์มาโฟกัสที่หน้าจอเบราว์เซอร์ ข้อมูลใหม่จะต้องปรับเปลี่ยนตามสด ๆ ทันทีโดยไม่ต้องกดรีเฟรชหน้าจอ (F5)
